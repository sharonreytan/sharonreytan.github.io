The Hard Parts I Wrote Myself

Everything here was built with strong teams — and these are just highlights of the pieces I wrote with my own hands.

The agent safety gate

Patent filed — implementation mine

How do you let an AI agent act inside a live, authenticated session — without trusting it? The agent never touches the browser: an allowlist gate validates every action, a deterministic executor runs it. Rejections become precise feedback — the gate grew coverage instead of just blocking.

The eval framework & observability

Measure before you steer. I built much of the eval framework that became the product's steering signal: full agent sessions in a website simulator, scored against static ground truth, with scenario and consistency runs. It proved the first agents at ~98% precision/recall — and justified killing that approach on cost anyway.

The xpath generator

Data acquisition lives or dies on selectors. The generator crafts xpaths that survive layout changes yet never grab the wrong element — the code-generation pipeline runs on it. Proof, live:

No ids, no positional indexes — anchored to meaning, so it survives redesigns. Hover me: it selects this card's own title on this very page.

The Elasticsearch library & the sync engine

The org-wide Elasticsearch library — a filter language that let any engineer query an unforgiving datastore safely — and the sync engine's hardest internals: a per-ID lock manager and partial document updates in Elasticsearch — a mechanism that didn't exist; we invented it.
